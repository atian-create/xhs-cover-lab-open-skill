# Benchmark And Safety Rules

Use this file when the user wants to learn from viral covers or adapt a
reference cover.

## Benchmark Output

If public benchmarks are available, return only a small first batch:

- 3-5 examples
- direct note links when possible
- visible metrics: likes, saves/collections, comments
- cover URL or thumbnail when possible
- cover type label
- one-second click reason
- learnable structure
- non-copyable resource

Do not start with 10-20 references unless the user asks for a visual wall,
50-cover study, or 100-cover study.

## Learnable vs Non-Copyable

Always separate:

1. Structure worth learning
   - title placement
   - keyword type
   - split screen
   - proof scene
   - information density
   - comment prompt
   - save reason

2. Resource that cannot be copied
   - face
   - room
   - authority title
   - city/store access
   - expensive production
   - personal life event
   - camera confidence
   - speaking ability
   - existing audience

3. Ordinary-user adaptation
   - what the user can do with their own material this week

## Currentness

If a reference is old, call it historical style reference instead of current
benchmark.

If no date is available, avoid saying the style is currently effective. Say:

> 这只能作为封面结构参考，不代表它现在仍然是最新爆款。

## Metrics Interpretation

Likes:

- show broad appeal or emotional resonance
- can be inflated by drama, event, or platform push

Saves / Collections:

- usually indicate useful value, checklist value, tutorial value, or planning
  value
- stronger for knowledge cards, travel guides, tool lists, food routes

Comments:

- show discussion, confusion, debate, question demand, or interaction design
- useful for interaction prompt covers and emotional topics

Save-like ratio:

- high save rate: users want to keep it
- high likes but low saves: may be emotional, funny, beautiful, or event-driven

## Do Not

- Do not copy exact title, layout, creator face, logo, brand asset, or private
  material.
- Do not invent metrics.
- Do not claim a cover is fake without evidence.
- Do not tell beginners to imitate high-production covers without warning.
- Do not treat "viral" as automatically worth learning.
