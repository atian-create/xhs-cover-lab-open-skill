# Cover Brief Contract

Use this file when the user wants to make a cover, send a prompt to a designer,
or prepare for image generation.

## Required Fields

1. Platform
   - Xiaohongshu / RedNote unless otherwise specified

2. Aspect ratio
   - default: vertical 3:4 or 4:5

3. Cover archetype
   - choose one primary type from `cover_archetypes.md`

4. Target audience
   - who is supposed to stop and click

5. Cover promise
   - what the viewer gets after clicking

6. Exact on-cover text
   - title
   - subtitle
   - small labels
   - bottom note if needed

7. Main visual subject
   - face, room, tool screenshot, food/place, product, text card, or clean card

8. Layout
   - title position
   - image position
   - proof scene
   - modules/cards
   - stickers/arrows/highlights if needed

9. Color and font direction
   - high contrast
   - readable Chinese text
   - avoid one-note palettes

10. Proof
   - what makes the claim believable

11. Avoid
   - fake metrics
   - fake screenshots
   - copied creator face/logo
   - crowded text
   - unrepeatable style

## Quality Gate

Before returning the brief, check:

- Can the topic be understood within one second?
- Is the biggest text readable?
- Does the cover have a click/save/comment reason?
- Does the visual match the user's real material?
- Can the account repeat this style for at least 10 posts?
- Is the cover adapted, not copied?
